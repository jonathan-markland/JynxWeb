AudioWorklet + WASM in Shared Memory + coi-serviceworker

Sound generation on AudioWorklet (2nd javascript "thread" instance).

Running of WASM program on the AudioWorklet (as the emulator would be).

Hosting the WASM memory in a SharedArrayBuffer, which has ramifications on the C program's linker, since a SHARED flag exists in the WASM executable to consent to the WASM memory being held in a JS SharedArrayBuffer.

Accessing the WASM memory via a Float32Array alias over the SharedArrayBuffer. This is a model for bi-directional data access, and would be used for the emulator keyboard/screen.
